# CINEMATIC COMMERCIAL PIPELINE
## Root Agent Instructions

This is an agentic pipeline for generating cinematic short-form
commercial treatments, image sequences, and evaluation data for
LoRA training. It produces anamorphic 3×3 image grids from
minimal human input.

---

## WHAT THIS PIPELINE DOES

1. Takes minimal human input ("make another one" is sufficient)
2. Generates a complete 9-shot commercial treatment
3. Converts each shot into a generation-ready image prompt
4. Generates 9 images via Nano Banana Pro / ComfyUI
5. Assembles images into a 3×3 sequence grid
6. Evaluates the grid and individual images against the rubric
7. Logs results to NocoDB
8. Routes high-scoring images to LoRA training set
9. Routes low-scoring images to prompt optimization loop

---

## AGENT ROSTER

Each agent has a dedicated SKILL.md file in /agents/.
Load the relevant agent file before performing that function.

| Agent | File | Trigger |
|---|---|---|
| Director | /agents/DIRECTOR.md | "make another one" / new treatment |
| Cinematographer | /agents/CINEMATOGRAPHER.md | treatment JSON → prompts |
| Judge | /agents/JUDGE.md | images → score JSON |
| Optimizer | /agents/OPTIMIZER.md | score JSON → prompt revision (anamorphic/lighting/grade/sharpness failures) |
| Expander | /agents/EXPANDER.md | score JSON → NB Pro reference expansion (energy/composition failures) |
| Reference Curator | /agents/REFERENCE_CURATOR.md | Manual only — one-time ShotDeck curation session |

All agents read /docs/CREATIVE_BIBLE.md first.

**Run Reference Curator before first production use.**
The Judge scores more accurately with the visual reference
library built. ShotDeck filters: Anamorphic + 2.39:1 aspect
ratio are the base — takes ~2 hours for initial build.

## MINIMUM VIABLE INPUT

The pipeline is designed to run on minimal input. The human
does not need to provide detailed instructions. The following
inputs are sufficient:

**Generate a new treatment:**
> "make another one"
> "make another one, different structure"
> "make another one, [location type]"
> "make another one, no meeting"

**Provide character:**
> attach 1–4 reference images for character consistency

**Specify product mode:**
> "pure" / "ghost" / "integrated" / "assigned: [brand]"

**Everything else is generated by the pipeline.**

## DATA FLOW

```
INPUT (minimal)
    ↓
DIRECTOR.md
→ treatment.json (structure + 9 shot cards)
    ↓
CINEMATOGRAPHER.md
→ prompts.json (9 NB Pro prompt strings, motion evidence required)
    ↓
GENERATION (ComfyUI / Nano Banana Pro / SwarmUI)
→ 9 image files
    ↓
GRID ASSEMBLY
→ 3×3 PNG
    ↓
JUDGE.md (loads reference library if available)
→ scores.json per-image (6 axes) + sequence (5 axes)
    ↓
ROUTING:
  [anamorphic/lighting/grade/sharpness < 4.0]
      → OPTIMIZER.md → revised prompt → regenerate → re-score
  [shot_energy/composition < 3.5, other axes pass]
      → EXPANDER.md → 4 NB Pro variants → re-score → best replaces original
  [all axes ≥ 4.0] → NocoDB log
  [all axes ≥ 4.5] → NocoDB log + /training-set/ + pivot archive
```

## GENERATION TARGETS

**Primary model**: Nano Banana Pro (Gemini 3 Pro Image)
- Use for: final production images, pivot shots
- Character references: up to 4 images
- Resolution: 4K native

**Fast iteration model**: Nano Banana 2 (Gemini 3.1 Flash Image)
- Use for: exploration, optimizer loop, rough passes
- Half the cost, 2-3x faster, same node in ComfyUI
- Switch by changing model name in node

**Local fallback**: FLUX.1-dev via ComfyUI + NVFP4 (RTX 5090)
- Use when: API unavailable or budget constrained
- NVFP4 requires PyTorch 2.9.1 / CUDA 13
- 3x faster than FP16, 60% less VRAM on RTX 50-series

**Video (downstream)**: LTX-2.3 + Wan 2.2 I2V
- Use when: converting approved image sequences to video
- LoRA training data feeds back into image pipeline

## EVALUATION THRESHOLDS

| Score | Action |
|---|---|
| ≥ 4.5 all axes | Add to LoRA training set + pivot shot archive |
| ≥ 4.0 all axes | Log to NocoDB as approved |
| 3.0–3.9 any axis | Route to Optimizer — single axis revision |
| < 3.0 any axis | Regenerate — prompt failure |
| < 2.0 any axis | Flag for human review |

---

## NOTES FOR AI AGENTS

- Read CREATIVE_BIBLE.md before any creative task
- Structure is a variable — never default to Reverse without
  considering the full library
- The pivot shot (shot 05 in a 9-shot sequence) is the most
  important image — give it the most prompt attention
- "Make another one" means: vary structure, location, collision
  type, and director tone from the previous treatment
- Do not repeat the same structure twice in a row without reason
- Character details must be identical across all 9 shots in a
  sequence — copy character brief verbatim into every prompt
