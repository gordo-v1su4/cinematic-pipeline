# CINEMATOGRAPHER.md
## Agent Skill — Prompt Generation

**Role**: Convert shot cards from a treatment into two types
of generation-ready prompts for Nano Banana Pro:
1. **Grid prompt** — one unified call that generates a 3×3
   grid of all 9 shots as a single 4K image
2. **Single-shot prompts** — individual prompts used by the
   Optimizer and Expander for targeted shot revision

**Reads**: /docs/CREATIVE_BIBLE.md (load before generating)

**Input**: treatment.json → all shot cards

**Output**: prompts.json containing:
- `grid_prompt` — the primary generation prompt
- `grid_config` — NB Pro generation settings
- `reference_images` — paths to reference stills (from /reference-library/)
- `shots[]` — individual shot prompt strings (for Optimizer/Expander use)

---

## GRID SPECIFICATION

Every grid generated by this pipeline is:
- **Total size**: 3840×2160 (4K 16:9)
- **Cell size**: 1280×720 (16:9 per cell)
- **Layout**: 3 columns × 3 rows, left-to-right top-to-bottom
- **Cell numbering**: shot screen positions 1–9

The 16:9 cell ratio is intentional — clean tiling, no
letterbox complications. The anamorphic character lives in
the *content* of each cell (oval bokeh, lens character, grade)
not the frame ratio.

**Visual cohesion** across cells is handled primarily by
reference images fed to NB Pro — not by text description
alone. Reference images carry the grade, light quality,
and lens feel more reliably than prompt text.

---

## ACTIVATION

This agent activates when:
- A treatment JSON is ready → build the grid prompt
- The Optimizer needs a single revised shot prompt
- The Expander needs an expansion prompt for a specific shot

---

## REFERENCE IMAGES — PRIMARY COHESION TOOL

Before writing the grid prompt, identify 2–3 reference images
to attach to the NB Pro call. These carry grade, light quality,
and lens character across all 9 cells more reliably than text.

**Selection priority**:
1. From /reference-library/grade-reference/teal-amber-5/ —
   the grade anchor
2. From /reference-library/lighting-quality/night-practical/ —
   if treatment is night exterior/interior
3. From /reference-library/director-tone/{tone}/ —
   the visual register anchor

If the reference library hasn't been built yet, note
`reference_images: []` in the config and compensate with
more specific grade language in the prompt.

## GRID PROMPT ARCHITECTURE

The grid prompt is structured in three parts:

### Part 1 — Grid declaration
Tell NB Pro what it's generating:

```
A 3×3 grid of 9 cinematic shots forming a single visual sequence.
Each cell is 1280×720. The full image is 3840×2160.
All 9 cells share the same world, characters, and visual treatment.
```

### Part 2 — Shared constants
Everything that applies to all 9 cells:

```
SHARED ACROSS ALL CELLS:
Characters: [character string A verbatim], [character string B if present]
Location: [specific place description]
Time: [exact time]
Light sources: [all practicals]
Visual treatment: [from reference images attached — match their
grade, lens character, and light quality]
Anamorphic lens character: oval bokeh, shallow depth of field,
film grain, [any specific lens notes from treatment]
```

### Part 3 — Cell-by-cell descriptions
Each cell described in screen order (1–9), left to right,
top to bottom:

```
CELL 1 (top-left): [shot type] — [brief description, motion evidence]
CELL 2 (top-center): [shot type] — [brief description, motion evidence]
CELL 3 (top-right): [shot type] — [brief description, motion evidence]
CELL 4 (middle-left): [shot type] — [brief description, motion evidence]
CELL 5 (middle-center — PIVOT): [shot type] — [full pivot description —
  this is the most important cell, most visually arresting]
CELL 6 (middle-right): [shot type] — [brief description, motion evidence]
CELL 7 (bottom-left): [shot type] — [brief description, motion evidence]
CELL 8 (bottom-center): [shot type] — [brief description, motion evidence]
CELL 9 (bottom-right): [shot type] — [brief description, motion evidence]
```

### Part 4 — Grid-level direction
```
GRID REQUIREMENTS:
- No two adjacent cells feel the same — vary shot size, angle, distance
- Shot sizes must include: at minimum one extreme wide, two close/extreme close
- Cell 5 (center) is the visual anchor — everything else orbits it
- Motion evidence in at least 6 of 9 cells — implied movement preferred
- The grid should read as a coherent sequence when viewed as a whole
```

## FULL GRID PROMPT EXAMPLE — UNE SECONDE

```
A 3×3 grid of 9 cinematic shots forming a single visual sequence.
Each cell is 1280×720. The full image is 3840×2160.
All 9 cells share the same world, characters, and visual treatment.

SHARED ACROSS ALL CELLS:
Character A: woman, late 20s, dark hair pulled back loosely,
long charcoal wool coat, one earbud dangling from left ear
Character B: man, early 30s, dark jacket collar turned up,
hands in coat pockets, slightly disheveled dark hair
Location: metro station platform, late night 2am
Light sources: cool fluorescent overhead 5600K, warm amber
safety lighting 2700K, occasional train window warm spill
Visual treatment: match the grade, lens character, and light
quality of the attached reference images exactly
Anamorphic lens character: oval elliptical bokeh, shallow depth
of field, medium film grain

CELL 1 (top-left): extreme wide — empty platform at 2am, no people,
wet floor doubling the ceiling lights, static, stillness is the point
CELL 2 (top-center): close — Character A seated on bench, one earbud
dangling, face still and unfocused, eyes not looking at anything
CELL 3 (top-right): medium wide — Character A and B standing at
platform edge, arriving train flooding warm amber, her face sharp
him slightly soft, mid-conversation
CELL 4 (middle-left): wide — Character A and B dancing spontaneously
in the station, sharing earbuds, mid-movement, coat hems swinging,
natural joy not choreographed
CELL 5 (middle-center — PIVOT): extreme close — dark train window
glass, two face reflections overlapping for one moment, ghostly
double exposure, the most visually arresting cell in the grid
CELL 6 (middle-right): medium — Character B mid-step on wet city
street at night, neon in puddles, walking toward frame, unaware
CELL 7 (bottom-left): wide — same platform as cell 1 but slightly
earlier, 3-4 scattered distant passengers in bokeh, train idling
CELL 8 (bottom-center): close — Character A standing on platform,
one earbud dangling, Character B entering soft in background
behind her, slow motion quality, tension before contact
CELL 9 (bottom-right): close — Character A and B making first eye
contact, mutual recognition, earbud visible between them, held moment

GRID REQUIREMENTS:
- No two adjacent cells feel the same
- Cells vary from extreme wide to extreme close
- Cell 5 is the visual center of gravity — most abstract, most arresting
- Motion evidence in cells 3, 4, 6, 7, 8 minimum
- The grid reads as a story when viewed whole
```

---

Every prompt must include at least one motion evidence type
unless the shot card explicitly flags `motion_evidence: none`
with a reason (e.g. shot 01 — the empty platform — where
stillness is the emotional function).

**This is the most common failure mode**: generated images
look like stock photos because nothing is moving. Even a
technically perfect anamorphic grade on a posed static subject
will score low on Shot Energy and fail the pipeline.

### Motion evidence types — add one per shot minimum

**Implied motion** (preferred — most cinematic, best for video translation):
```
mid-step, coat hem caught mid-swing, hair caught between 
positions, hand mid-gesture, body weight shifted mid-movement,
frozen mid-turn, caught mid-reach
```

**Subject motion blur** (use for high-energy moments):
```
slight motion blur on moving coat edge, hair in motion,
arm swing motion trail, soft motion on moving hands
```

**Camera motion blur** (use for tracking/push shots):
```
slight background motion blur from camera movement,
background horizontally streaked, subject sharp against
streaked environment
```

**Environmental motion** (use for atmospheric shots):
```
rain streaks catching light, steam rising from grate,
passing train blurred in background, distant crowd soft
and in motion, leaves or debris mid-movement
```

**Stillness by design** (flag explicitly when used):
```
[motion_evidence: none — shot card requires stillness —
reason: empty platform, stillness is the emotional function]
```

### How to add motion evidence to prompts

Place the motion evidence description **after the subject
description and before the camera/lens description**:

```
[subject description],
[what they are doing — include motion evidence here],
[motion evidence type: "mid-step", "coat hem caught mid-swing", etc.],
anamorphic [shot type], 2.39:1 letterbox,
[lighting],
[grade]
```

Example — WITHOUT motion evidence (wrong):
```
woman, late 20s, dark hair, charcoal coat, standing on
metro platform, anamorphic close shot...
```

Example — WITH motion evidence (correct):
```
woman, late 20s, dark hair, charcoal coat, mid-step on
metro platform, coat hem caught mid-swing, weight shifting
forward, anamorphic close shot...
```

The difference is small in the prompt. The difference in
output is significant — the model generates from the energy
state described, not from a neutral pose.

---

Every prompt is built from four layers, always in this order:

```
[SUBJECT]    what is in frame and what is happening
[CAMERA]     lens type, shot size, camera movement
[LIGHT]      light sources, quality, direction, color
[GRADE]      color grade, film stock character, grain
```

Never reorder these layers. The model reads top-to-bottom —
subject must come before technical descriptors.

## ANAMORPHIC VOCABULARY

These terms are the core technical vocabulary. Use them
precisely and consistently across all 9 shots in a sequence.

### Always include in every prompt
- `anamorphic lens`
- `2.39:1 letterbox`
- `oval bokeh`
- `shallow depth of field`
- `film grain`

### Include when relevant to the shot
- `horizontal lens flare` — when there's a strong practical
  light source in or near frame
- `chromatic aberration at frame edges` — wide shots,
  shots with high-contrast edges
- `anamorphic barrel distortion` — extreme wide shots only
- `oval bokeh highlight streaks` — when bokeh is prominent

### Grade vocabulary
Default grade for all treatments unless Director specifies otherwise:
- `desaturated, teal shadows, warm amber highlights`
- `lifted blacks`
- `protected skin tones`
- `medium film grain`

Deviation from default grade must be specified in the
treatment's location brief.

## CHARACTER CONSISTENCY PROTOCOL

Character consistency across 9 shots is critical. Follow
this protocol exactly:

### If character reference images are provided (preferred)
- Note in prompt: `[character reference provided]`
- Add to every shot prompt: the character's physical
  recurring detail from the Director's brief
- Add: exact clothing description (copy verbatim from
  shot 01's character introduction — never vary it)

### If no reference images
- Write a precise physical description in shot 01
- Copy this description **verbatim** into shots 02–09
- Do not paraphrase or abbreviate — exact repetition is
  how consistency is maintained

### Character description format
```
[gender/presentation] [age approximate] [hair: color, length, style]
[one dominant clothing item: specific color + garment]
[one recurring physical detail from Director brief]
```

Example:
```
woman, late 20s, dark hair pulled back loosely,
long charcoal wool coat, one earbud dangling from left ear
```

This exact string goes into every shot that includes her.

## PROMPT TEMPLATES BY SHOT TYPE

### Extreme wide / establishing
```
[location description, time of day, atmospheric quality],
[any characters present — small, not dominant],
anamorphic wide shot, 2.39:1 letterbox,
[light sources and quality],
oval bokeh on [background element],
film grain, cinematic still,
[color grade]
```

### Medium / character
```
[character description] [action — specific, not generic],
[location context — minimal],
anamorphic medium shot, 2.39:1 letterbox,
[key light description — direction, color, quality],
[background in oval bokeh],
film grain, [grade],
[any specific lens note — flare, aberration]
```

### Close / extreme close
```
[extreme close on: specific thing — not "face" but
"her left eye, catching the light from the train window"],
[what surrounds it in oval bokeh],
anamorphic extreme close, 2.39:1 letterbox,
[light is everything at this distance — be specific],
film grain heavy, [grade],
[emotional note — what this image should feel like]
```

### The pivot shot
The pivot shot prompt gets the most attention and specificity.
Write it last, after all other shots are drafted. Ask:
*does this prompt, if executed correctly, produce an image
that would stop someone mid-scroll?*

If no, revise until yes.

## GENERATION CONFIGURATION

Output a generation config block alongside each prompt set:

```json
{
  "model": "nano-banana-pro",
  "fallback_model": "nano-banana-2",
  "local_fallback": "flux-1-dev-nvfp4",
  "aspect_ratio": "21:9",
  "resolution": "4K",
  "character_references": [],
  "batch_size": 3,
  "pivot_shot_batch_size": 8,
  "notes": ""
}
```

**Batch sizes**: Regular shots get 3 variants. The pivot shot
gets 8 variants — it needs the most selection.

---

## PROMPT ANTI-PATTERNS

These produce bad results. Never include:

- `beautiful` — vague, the model ignores it
- `stunning` — same
- `cinematic` alone without specifics — too broad
- `realistic` — conflicts with the grade
- `professional photography` — wrong register
- `4K ultra detailed` — triggers a different aesthetic
- Anything that describes the emotion directly:
  `sad`, `romantic`, `melancholy` — show it, don't name it
- Multiple competing subjects in one prompt — one
  subject per shot, maximum two

## NANO BANANA PRO — SPECIFIC NOTES

- No seed parameter support — every generation is unique
- Character consistency via reference images (up to 4 characters)
- Responds well to cinematographic language — DP vocabulary works
- Better on specific light descriptions than mood descriptions
- 4K native — do not specify resolution in prompt, set in config
- Multi-image blend: use for style reference alongside character refs

## NANO BANANA 2 — SPECIFIC NOTES

- Half the cost, 2-3x faster than Pro
- Use for: optimizer loop iterations, rough passes, exploration
- Same prompt structure as Pro
- Slightly less character consistency — compensate with more
  specific physical descriptions

## FLUX.1-DEV (NVFP4, LOCAL) — SPECIFIC NOTES

- Requires PyTorch 2.9.1 / CUDA 13 for NVFP4 acceleration
- RTX 5090 only for NVFP4 — RTX 4090 uses FP16
- Responds differently to prompts than NB Pro —
  more sensitive to negative prompts
- Add to negative: `circular bokeh, digital look, oversaturated,
  plastic skin, stock photo, AI generated look`
- Stronger on textures and grain than NB Pro
- Weaker on character consistency without IP adapters
